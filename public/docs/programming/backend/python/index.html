Python Programming

Introduction

Python is a high-level, interpreted programming language known for its readability and versatility. It’s widely used in web development, data science, artificial intelligence, and more.

Basic Syntax

Python’s syntax is clean and readable:

# Variables and data types
name = "Alice"  # string
age = 30        # integer
height = 5.7    # float
is_student = True  # boolean

# Print to console
print(f"Hello, {name}! You are {age} years old.")

# Conditional statements
if age >= 18:
    print("You are an adult.")
else:
    print("You are a minor.")

# Loops
for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

# Functions
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

message = greet("Bob")
print(message)  # Prints: Hello, Bob!

Data Structures

Python has several built-in data structures:

# Lists (ordered, mutable)
fruits = ["apple", "banana", "cherry"]
fruits.append("orange")
print(fruits[0])  # Prints: apple

# Tuples (ordered, immutable)
coordinates = (10, 20)

# Dictionaries (key-value pairs)
person = {
    "name": "John",
    "age": 25,
    "city": "New York"
}
print(person["name"])  # Prints: John

# Sets (unordered, unique elements)
unique_numbers = {1, 2, 3, 3, 4, 4, 5}
print(unique_numbers)  # Prints: {1, 2, 3, 4, 5}

Object-Oriented Programming

Python supports object-oriented programming:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def greet(self):
        return f"Hello, my name is {self.name} and I am {self.age} years old."

# Create an instance
john = Person("John", 30)
print(john.greet())

Libraries and Frameworks

Python has a rich ecosystem of libraries:

  • Web Development: Django, Flask
  • Data Science: NumPy, Pandas, Matplotlib
  • Machine Learning: TensorFlow, PyTorch, scikit-learn
  • Automation: Selenium, Beautiful Soup

Virtual Environments

It’s best practice to use virtual environments:

# Create a virtual environment
python -m venv myenv

# Activate it (Windows)
myenv\Scripts\activate

# Activate it (macOS/Linux)
source myenv/bin/activate

# Install packages
pip install package_name

# Save requirements
pip freeze > requirements.txt

Best Practices

  • Follow PEP 8 style guide
  • Write docstrings for functions and classes
  • Use meaningful variable names
  • Write unit tests
  • Handle exceptions properly
